# STREAM PROBLEM - SSL Zertifikat

## Das Problem
- Website: `https://yourparty.tech` (HTTPS)
- Stream: `http://radio.yourparty.tech/...` (HTTP)
- **Browser blockiert**: Mixed Content (HTTP auf HTTPS-Seite)

## Warum HTTP nicht funktioniert
Wolfgang's Frage: "Wenn das nur HTTP ist, wie komme ich dann von der WWW darauf?"
**Antwort**: GAR NICHT! Browser blockieren HTTP-Inhalte auf HTTPS-Seiten aus Sicherheitsgründen.

## Die richtige Lösung
SSL-Zertifikat für `radio.yourparty.tech` muss gültig sein.

**Aktueller Stand**:
- Zertifikat: `/etc/letsencrypt/live/npm-29/fullchain.pem`
- Domain: Wahrscheinlich nur `yourparty.tech` (nicht `radio.yourparty.tech`)

**Was wir brauchen**:
1. Neues Let's Encrypt Zertifikat für `radio.yourparty.tech`
2. ODER Wildcard-Zertifikat `*.yourparty.tech`

**Wo das gemacht wird**:
- Nginx Proxy Manager (NPM) auf CT 103
- Web-Interface: Vermutlich `http://192.168.178.103:81`

## Nächste Schritte
1. NPM Web-Interface öffnen
2. SSL-Zertifikat für `radio.yourparty.tech` anfordern
3. Proxy Host 10 (radio.yourparty.tech) aktualisieren
4. Stream-URL zurück auf HTTPS ändern
