# SSL Certificate Fix - Status

## Was das Problem ist
- Stream-URL: `https://radio.yourparty.tech/listen/radio.yourparty/radio.mp3`
- SSL-Zertifikat: Für "vaultwarden.yourparty.tech" (FALSCH!)
- Browser blockiert: `ERR_CERT_COMMON_NAME_INVALID`
- **Ergebnis**: Keine Musik hörbar

## Was ich versucht habe
1. ❌ HTTP statt HTTPS → Mixed Content (blockiert)
2. ❌ Proxy über yourparty.tech → 503 Fehler
3. 🔄 **JETZT**: Neues SSL-Zertifikat anfordern

## Die richtige Lösung
**Certbot** in Nginx Proxy Manager Container ausführen:
```bash
certbot certonly --webroot \
  -w /data/letsencrypt-acme-challenge \
  -d radio.yourparty.tech \
  --email wolf@yourparty.tech \
  --agree-tos --non-interactive
```

**Dann**: Nginx Config für radio.yourparty.tech aktualisieren:
```nginx
ssl_certificate /etc/letsencrypt/live/radio.yourparty.tech/fullchain.pem;
ssl_certificate_key /etc/letsencrypt/live/radio.yourparty.tech/privkey.pem;
```
